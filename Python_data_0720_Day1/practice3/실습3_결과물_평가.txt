실습 3 - 결과물 평가

- asyncio.Semaphore(MAX_CONCURRENT=10)로 동시 요청 수를 제한해서 서버에 한 번에 몰리지 않게 요청 수 제어를 구현
- async with asyncio.timeout(REQUEST_TIMEOUT)으로 요청 하나하나에 타임아웃(3초) 상한을 설정
- fetch_retry 함수에서 실패 시 지수 백오프(2 ** attempt초, 1→2→4초)로 최대 3회까지 자동 재시도하고, 마지막 시도까지 실패하면 실패 결과를 그대로 반환
- collect 함수에서 60개 요청을 동시에 실행하며, 하나가 실패해도 나머지 결과는 살아남도록 예외를 격리
- main 함수에서 성공/실패 건을 나눠 재시도까지 실패한 건(dead-letter)만 dead_letter.json에 별도로 저장

[잘한 점]
- 가이드의 step0~step6, 확장과제 코드를 하나하나 따라 작성하면서 각 라인에 주석을 달아 이해한 뒤 다음 단계로 넘어가는 방식으로 실습을 진행해서, 최종 실습3 코드의 전체 흐름을 자연스럽게 이해하며 완성할 수 있었다.
- FAIL_ONCE_ID = 5, attempt == 0 조건으로 5번 요청만 첫 시도에 일부러 실패하게 만들어서, 재시도 로직이 잘 동작하는지 직접 눈으로 확인하였다.
- time.perf_counter()로 시작 시간과 끝난 시간을 재서 걸린 시간을 계산하고, 비교 문구를 같이 출력해 비동기 처리가 실제로 얼마나 빨라졌는지 숫자로 확인할 수 있게 만들었다.

[아쉬운 점 / 어려웠던 점]
- async with sem: 안에 또 async with asyncio.timeout() 이 겹쳐 있는 구조를 이해하는 데 시간이 걸렸다.
- asyncio는 여러 코어로 동시에 돌리는 게 아니라, 기다리는 동안 잠깐 다른 작업에게 순서를 양보하는 방식 이해가 어려웠다.
