실습 1 - 결과물 평가

- 제너레이터(read_logs)로 web_logs.csv를 한 줄씩 읽어, 파일 전체를 메모리에 올리지 않고 스트리밍 방식으로 집계
- for 루프 방식과 functools.reduce + fold 패턴 두 가지 방식으로 동일한 집계를 구현
- 상태코드 / 경로 / 시간대 / IP별 건수와 5xx 오류율을 계산하고, checkpoint()로 총 건수·오류율 기준을 자동 PASS/FAIL 검증
- tracemalloc으로 readlines() 방식과 제너레이터 방식의 최대 메모리 사용량 비교

[잘한 점]
- 주어진 guide의 step0~step6 코드를 파일 하나하나 따라 작성하고 읽어보면서, 각 라인에 주석을 달아가며 이해한 뒤 다음 단계로 넘어가는 방식으로 실습을 진행해서 최종 실습1 코드의 전체 흐름을 수월하게 따라갈 수 있었다.
- guide의 확장 과제  코드도 구현해보면서, 왜 제너레이터가 메모리 효율적인지에 대한 이해를 높였다.
- for loop 결과와 reduce 결과를 교차 검증해 total이 일치 확인하여 각 방식을 비교해 이해하였다.

[아쉬운 점 / 어려웠던 점]
- reduce(fold, iterable, init)의 동작 방식을 이해하는 데 어려움이 있었다. fold(acc, row)가 매번 return한 acc가 다음 줄 처리의 acc로 그대로 이어진다는 흐름이 for 루프처럼 눈에 바로 보이지 않아서 이해하는데 시간이 오래 걸렸다.
- get_traced_memory()가 바이트 값을 출력해 자릿수가 너무 많아 두 버전의 메모리 차이를 한눈에 비교하기 어려웠다. MB 단위로 변환하기 위해 1024로 두 번(바이트→KB→MB) 나눈 값을 비교하니 차이 비교가 쉬웠다.
