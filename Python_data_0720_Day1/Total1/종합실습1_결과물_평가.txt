종합실습 1 - 비동기 ETL 파이프라인 결과물 평가

- Extract: 비동기로 데이터 모으기, 동시에 너무 많이 안 보내게 제한, 실패하면 재시도
- Transform: Pydantic으로 데이터 검사해서 이상한 것(음수 가격 등) 걸러내기
- Load: DataFrame으로 만들어서 CSV, Parquet 파일로 저장
- Orchestrate: run() 함수 하나로 위 3단계 순서대로 실행
- pytest 테스트 6개 전부 통과, ruff도 통과함

[잘한 점]
- practice 코드를 바탕으로 Extract/Transform/Load/Orchestrate를 함수 단위로 나눠서, 이해하면서 구현할 수 있었다.
- Pydantic으로 카테고리 소문자 변환, 음수 가격 거부 같은 검증 규칙을 직접 만들며 이해하였다.
- 몇 개 id는 의도적으로 첫 시도에 실패하게 만들어서, 재시도가 진짜 되는지 확인했다.

[아쉬운 점 / 어려웠던 점]
- asyncio.Semaphore가 이벤트 루프에 묶인다는 개념 이해가 어려웠다.
- 비동기 코드가 실제로 어떤 순서로 실행되는 건지 코드를 읽으며 이해하기 어려워 오래걸렸다.
- practice1, 2, 3에서 함수 하나씩 실습했을 때는 각각 이해하기 어렵지 않았는데, 종합 실습에서 다 모아서 하나의 파이프라인으로 연결하니까 어렵게 느껴졌다.
